크롤링을 통해 html 파일을 수집해두었는데 규칙은 다음과 같아
1. 수집 기준일은 배송은 tms_delivery.csv의 주문번호 1째부터 8째 자리까지, 수거는 tms_return.csv의 반품번호  2째부터 9째 자리까지 YYYYMMDD 형태로 저장됨
2. tms_delivery.csv및 tms_return.csv 파일의 상세보기 필드에 있는 경로 앞에  현재 폴더 대신 /var/www/better 를 붙이면 실제 파일 위치임.
3. 상세보기 필드에 저장된 파일의 상대경로에 있는 파일들은 모두 주문번호 또는 반품번호에 확장자 html 을 붙인 구조이기 때문에 날짜 정보만으로 파일들이 수집 되었는 지 판단이 가능해.

이 규칙을 이용해서 다음과 같은 점검 결과를 출력하는 python 스크립트를 작성해줘.
1. 점검할 날짜 의 기본값은 어제 이고 입력 받을 수 있게 함
2. 요약데이터 tms_shipping.csv 또는 tms_return.csv에 점검할 날짜의 데이터가 몇 개가 기록 됐는 지 확인
3. /var/www/better 아래에 점검할 날짜의 html 파일들이 몇 개가 기록됐는 지 확인
4. 위 2,3의 건수를 쉽게 비료할 수 있도록 출력
5. 위 건수가 어느 한 쪽이 0이면 수집이 안됐다는 메세지와 함께 해당 건수 출력
6. 위 두 종류의 건수가 맞는 지 확인할 수 있도록 출력
